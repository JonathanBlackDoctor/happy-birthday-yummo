## 2. 기술 스택 (확정)

### 2.1 핵심 결정

```
엔진: 자체 React/TypeScript 비주얼 노벨 엔진 (자체 개발)
  - 이유: GitHub Pages 정적 호스팅 최적, 모바일 반응형 자유, UI 100% 커스터마이징,
          한국어 폰트/이모지 처리 자유, Tyrano/Renpy 대비 가벼움.
빌드: Vite + TypeScript
스타일: Tailwind CSS + CSS Variables (디자인 토큰)
사운드: Howler.js (BGM/SFX 풀링)
상태 관리: Zustand (가벼움, 직렬화 쉬움 → 저장 슬롯 호환)
시나리오 포맷: TSX 또는 JSON 스크립트 (예: `scenes/ch01_meeting.json`)
이미지 포맷: WebP (우선) + AVIF (폴백) + PNG (최후 폴백)
영상 포맷: MP4 (H.264) + WebM (VP9 폴백)
폰트: Pretendard (woff2 서브셋)
저장: localStorage (자동저장 1슬롯 + 수동저장 6슬롯)
빌드 산출물: 정적 HTML/JS/CSS + 자산 → GitHub Pages
CI/CD: GitHub Actions (push → 빌드 → 테스트 → Pages 배포)
```

### 3.2 분기 시스템

- **호감도**: 숨김 수치(0~100). 히로인 5명 각각 별도. UI에는 표시 안 됨.
- **선택지마다 +1~+5, -1~-5 변동** ← 본 표기법은 톤 매트릭스 도입으로 [CHOICE: tone=..., key=...]로 전환 예정 (CONVENTIONS §3.7).
- **카톡 답장 속도**: 답장이 너무 늦으면 호감도 -3 (거절 엔딩 핵심 트리거)
- **결정 챕터**: Ch.5 종료 시점에 호감도 1위가 메인 루트 결정. 동률 시 마지막 +값 받은 쪽.
- **트루엔딩 조건**: 단일 히로인 호감도 ≥80 + 특정 키 선택지 3개 통과
- **노멀엔딩**: 호감도 50~79
- **배드엔딩**: 호감도 ≤49 (또는 거절 트리거 발동 시 별도 분기)

### 3.3 시스템 기능 (모두 포함)

- ✅ 텍스트 표시/스킵/오토 (3단계 속도)

- ✅ CG 갤러리 (해금된 이벤트 CG만)

- ✅ 엔딩 리스트 (해금된 엔딩 ?/15)
- ✅ 카톡 메신저 시스템 (히로인 5명 모두 — 거절 엔딩 핵심 + 일상 호감도 관리)

### 5.2 텍스트박스 (하단 고정형)

- 위치: 화면 하단 고정 (margin-bottom: 40px)
- 너비: 화면의 86%, 좌우 중앙
- 높이: 28% (자동조정)
- 화자명 박스: 좌측 상단 살짝 튀어나옴 (민트 액센트)
- 폰트: Pretendard, 22px, line-height 1.6
- 텍스트 등장: 글자별 타이핑 (속도 3단계)
- 다음 표시: 우측 하단 깜빡이는 ▼

### 5.3 Gemini 워터마크 처리 (확정 방식 a)

**모든 이미지를 텍스트박스가 우측 하단을 덮도록 디자인.**
- 텍스트박스 우측 끝이 이미지 오른쪽 끝까지 도달
- 텍스트박스 높이 28% → 이미지 하단 28% 가림 → 워터마크는 보통 하단 5%에 있으므로 가림
- **CG 갤러리에서 풀스크린으로 볼 때**: 별도로 자른(crop) 버전 사용 → `public/img/cg/{name}_full.webp` (워터마크 영역 잘라낸 버전)
