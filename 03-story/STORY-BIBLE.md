## 6. 핵심 분기 룰

### 6.1 호감도 시스템

- 5개 변수: `H1~H5`, 각 0~100, 시작 0 (types.ts/STATE-SCHEMA 정합 — 옛 `affection_H1~H5` 명명은 2026-04-30 갱신)
- 선택지마다 +1~+5 또는 -1~-5
- 챕터 종료 시 어떤 히로인 호감도가 ≥40인지에 따라 다음 챕터 분기

### 6.2 결정 시점 (Lock-in)

- **Ch.5 종료 시점에 1위 히로인이 메인 루트로 확정**
- 동률 시 마지막에 +값 받은 히로인 우선
- 모든 호감도 < 30이면 → "혼자 여름방학" 노멀 단독 엔딩 (`END_SOLO_SUMMER`, 16번째 엔딩 확정 — 2026-04-28)

### 6.3 엔딩 분류

| 등급 | 조건 | 비고 |
|---|---|---|
| 트루 | 단일 히로인 ≥80 + 키 선택지 3개 통과 (H4 한정 **≥70** — 2026-04-30 Step 4 완화) | 영상 컷 포함 |
| 해피 | 단일 히로인 ≥60 + 키 선택지 2개 통과 | 정적 CG + 텍스트 |
| 노멀 | 단일 히로인 ≥40 | 텍스트 위주 |
| 배드 | 단일 히로인 ≤39 + 그 루트로 진입 | |
| **거절 (H4 전용)** | H4 루트 진입 시 (답장 지연 ≥2회) **OR** (aff <60) | **카톡 풀스크린 엔딩** |
| **단독 (16번째)** | 모든 호감도 <30 → `END_SOLO_SUMMER` "혼자 여름방학" | 노멀 분류 |

총 16개 엔딩 (히로인 5명 × 4단계 - H4 BAD 거절 흡수 + 단독 노멀 엔딩 1 = 16개)
※ H4는 BAD가 별도 존재하지 않고 거절(REJECT)이 흡수: 답장 지연 ≥2회 우선 평가, 그 외에 aff <60도 거절 라우팅. 자세한 평가 순서는 `BRANCH-GRAPH.md` §6.1 참조.
※ 단독 노멀 엔딩(모든 호감도 <30 시 "혼자 여름방학")은 2026-04-28 추가 확정.
실제 분포는 `BRANCH-GRAPH.md`에서 정확히 산출.
