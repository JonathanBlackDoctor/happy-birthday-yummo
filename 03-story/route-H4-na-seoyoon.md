# 03-story/route-H4-na-seoyoon.md

> ⚠️ **이 루트는 거절 카톡 엔딩의 핵심 무대**. MASTER-PLAN §4.3과 H4-na-seoyoon.md §6의
> 정확한 카톡 텍스트 + 8단계 연출을 그대로 따른다.

## Ch.6 "나서윤 분기" (약 15~20분, 분기에 따라 다름)

### 진입 조건
- `main_heroine === "H4"`

### 무대
- 계명대 캠퍼스 인근 또는 동성로, 카톡 모달 (거절 시)

### 비트 (분기 평가까지 공통)
1. **데이트 약속 잡기** — Ch.5 회식 후 H4와 카톡으로 약속 잡음
2. **답장 미니게임** — 1~2회 더 발생 (`late_reply_count` 누적 가능)
3. **데이트 시퀀스** — 계명대 캠퍼스 인근 또는 동성로
4. **이벤트 CG 재활용 또는 신규** — 호감도에 따라 CG 다름

### 분기 평가 (Ch.6 데이트 직후, BRANCH-GRAPH §6.1 동일 순서)

```
aff = state.flags.H4
late = state.flags.late_reply_count
key_count = countKeyChoices(state, "H4")

if late >= 2 → END_H4_REJECT (즉시, 호감도/순위 무관)
elif aff < 70 → END_H4_REJECT (BAD 자리 흡수)
elif aff < 95 → END_H4_NORMAL              // 모닥불 +25 흡수 임계 70→95
elif key_count >= 3 → END_H4_TRUE
else → END_H4_NORMAL  // aff ≥70 + 키 부족 fallback
```

> ⚠️ 평가 순서가 중요. 예: `late=1, aff=80, key=3` → late <2 통과 → aff <60 false → aff <80 false → key ≥3 → **TRUE**. (이전 의사코드의 `late == 0` 조건은 오류였음 — `late < 2` 즉 0 또는 1이면 정상 분기)

## END_H4_REJECT — "답장이 늦어서" (거절 카톡 엔딩) ⚠️ 핵심

### 시기
데이트 다음 날 아침.

### 무대
- 윤모의 자취방 (어두운 새벽 → 카톡 풀스크린)

### ⚠️ 정확한 카톡 텍스트 (변경 금지)

```
답장이 너무 늦어서 미안해ㅠㅠ
그날 만나서 얘기하고 시간 잘 보냈는데
더 진행하기엔 무리가 있을거 같아..
좋은 인연 만나길 바랄게 🥺🥺
```

### 8단계 연출 (정확히 이 순서)

1. **페이드 인** — 카톡 화면 풀스크린으로 전환 (검은 배경에서 페이드 인)
2. **BGM 전환** — `bgm_sad.mp3` 재생 시작 (잔잔한 어쿠스틱)
3. **카톡 알림 SFX** — 1회 재생 (`sfx_katalk_notify`)
4. **타이핑 인디케이터** — 나서윤의 프로필에 "입력 중..." 1.5초간 표시
5. **메시지 한 줄씩 타이핑** — 위 4줄 카톡을 한 줄당 0.8초 간격으로 페이드 인
6. **2초 정지** — 마지막 🥺🥺 등장 후 정적
7. **검은 페이드 아웃**
8. **타이틀 카드** — 검은 배경 + 흰 글자: "BAD ENDING — 답장이 늦어서"

### 영상 재생
- `video_reject_seoyoon.mp4` (5~7초): 비 내리는 창문 + 카톡 화면 비춤

### 엔딩 후 처리
- 엔딩 리스트에 "END_H4_REJECT" 해금
- "엔딩 리스트에 해금되었습니다" 토스트
- 메인 메뉴 또는 챕터 선택 화면으로 복귀 옵션

### BGM
`bgm_sad.mp3`

### 윤모 모놀로그 (선택, 짧게)
연출 중 또는 직후 1~2줄만 — 너무 길면 거절의 임팩트 손상.

```
[구윤모 모놀로그] (창밖 비) ...
[구윤모 모놀로그] 답장이 늦었던 게... 그렇게 큰 일이었나.
```

## END_H4_TRUE — "성서의 봄" (트루)

### 진입 조건
- `H4 >= 95` AND `key_count >= 3` AND `late_reply_count < 1` (BRANCH-GRAPH §6.1, 모닥불 +25 흡수 임계 70→95 + 명명 정합 — 옛 `affection_H4` → `H4` types.ts 정합)

### 시기
2026년 6월 말, 계명대 약대 시험 종료 직후.

### 무대
- 계명대 학생회관 앞 또는 캠퍼스 벚꽃길 (낮)

### 비트
1. 윤모가 H4의 시험 종료 응원하러 같은 캠퍼스 학생회관 앞으로
2. 학생회관 앞에서 H4가 가방 어깨에 메고 나옴
3. **이벤트 CG: cg_seoyoon_true** — 햇살, 부드러운 미소 (CG 영문 디스크립터는 W3 라운드에서 영남대→계명대 갱신 예정)
4. **영상 재생: video_true_seoyoon.mp4** (7초)
5. H4의 짧은 한 마디 — "답장 빠르게 해줘서 고마워, 윤모야"
6. 윤모 모놀로그: "이름을 부르는 게... 이렇게 무겁고 가벼울 줄"
7. 엔딩 타이틀: "나서윤 트루 엔딩 — 성서의 봄"

### BGM
`bgm_climax.mp3`

## H4 BAD 없음

H4는 BAD 엔딩 자리를 거절 카톡 엔딩이 흡수. (BRANCH-GRAPH §2 참조)
이는 사용자의 핵심 요구사항이며, 거절 카톡 시뮬레이션 자체가 메커니즘 핵심.

## 답장 미니게임 명세 (UI 시드)

상세는 `06-engine/ARCHITECTURE.md` + `05-ui-design/UI-SPEC.md` 참조. 시나리오 작성 시 활용 룰:

- H4가 메시지 보낼 때마다 화면 우측 상단 "답장하기" 알림 토스트
- 토스트 클릭 = 카톡 모달 열림 → 답장 선택지
- 토스트 **3초** (2026-05-09 15→3초 단축, STORY-BIBLE §7.1) 무시 → `late_reply_count++` + 즉시 RejectEnding 진입 (호감도 -3은 폐기 — 즉시 패배 룰로 단일화)
- 답장 선택지 중 부적절한 것 선택 → `late_reply_count++` 가능 (시나리오 명시)
